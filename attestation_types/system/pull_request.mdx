title: "System type: Pull request"

---

`system:pull_request` is for attesting that a code change came in through a pull request on GitHub, GitLab, Bitbucket, or Azure DevOps. The Kosli CLI fetches pull-request data from your git provider's API and records the URL, state, author, approvers, and commits.

## Versions

| Version | Status | Notes |
|---|---|---|
| `v1` | Current | Initial release. Default compliance is structural only ‚Äî attach a rego policy to enforce thresholds like "at least one approver". |

## What the attestation accepts

A pull-request attestation carries:

- **`git_provider`** *(required)* ‚Äî one of `github`, `gitlab`, `bitbucket`, `azure`
- **`pull_requests`** *(required)* ‚Äî an array of pull requests associated with the change. Each entry includes:
  - `url` ‚Äî the PR URL
  - `state` ‚Äî open, merged, closed
  - `merge_commit` ‚Äî the merge commit SHA (where applicable)
  - `approvers` ‚Äî list of users who approved
  - `author`, `title`, `head_ref`, `created_at`, `merged_at` ‚Äî PR metadata
  - `commits` ‚Äî the list of commits the PR carries
- **`evaluation_result`** *(optional)* ‚Äî the JSON output of `kosli evaluate` with a rego policy. When supplied, drives compliance.
- **Policy file attachment** *(required when `evaluation_result` is supplied)* ‚Äî the `.rego` file is uploaded to the audit trail.

## Compliance behaviour

Default: **structural only** ‚Äî the attestation is compliant if `pull_requests` is present (the array can be empty). Kosli records the PRs as a fact; whether "no PR" or "one with no approvers" is acceptable is your call.

With a rego policy attached: the policy's `allow` flag drives compliance. A typical policy enforces "at least one PR exists and every PR has at least one approver":

```rego
package policy

import rego.v1

default allow = false

violations contains msg if {
    count(input.pull_requests) == 0
    msg := "no pull request was associated with the change"
}

violations contains msg if {
    some pr in input.pull_requests
    count(pr.approvers) == 0
    msg := sprintf("pull-request %s has no approvers", [pr.url])
}

allow if count(violations) == 0
```

## How to use it

The pattern is identical to SARIF ‚Äî see the [SARIF tutorial](/tutorials/attest_sarif) for the full walkthrough. The legacy `kosli attest pullrequest_*` commands (one per git provider) continue to work; see [`kosli attest pullrequest_github`](/client_reference/kosli_attest_pullrequest_github), [`kosli attest pullrequest_gitlab`](/client_reference/kosli_attest_pullrequest_gitlab), [`kosli attest pullrequest_bitbucket`](/client_reference/kosli_attest_pullrequest_bitbucket), and [`kosli attest pullrequest_azure`](/client_reference/kosli_attest_pullrequest_azure) for those paths.
