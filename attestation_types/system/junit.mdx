`system:junit` is for attesting test results parsed from JUnit-format XML. Most test runners (pytest, JUnit itself, Mocha with `mocha-junit-reporter`, Jest with `jest-junit`, Go's test runner via `go-junit-report`, and so on) produce JUnit XML; the Kosli CLI parses it into the structure this attestation accepts.

## Versions

| Version | Status | Notes |
|---|---|---|
| `v1` | Current | Initial release. Default compliance is structural only — attach a rego policy to enforce thresholds like "zero failures". |

## What the attestation accepts

A JUnit attestation carries:

- **`junit_results`** *(required)* — an array of test-suite results. Each suite includes:
  - `name` — the suite name (typically the test class or file)
  - `failures` — count of test failures
  - `errors` — count of test errors
  - `skipped` — count of skipped tests
  - `total` — total tests
  - `duration` — wall-clock duration in seconds
- **`evaluation_result`** *(optional)* — the JSON output of `kosli evaluate` with a rego policy. When supplied, drives compliance.
- **Policy file attachment** *(required when `evaluation_result` is supplied)* — the `.rego` file is uploaded to the audit trail.

The CLI extracts the test-suite fields from a directory of JUnit XML reports.

## Compliance behaviour

Default: **structural only** — the attestation is compliant if `junit_results` is present and well-formed. Kosli does not impose an opinion on what counts as a "passing" test run; you do.

With a rego policy attached: the policy's `allow` flag drives compliance. A common policy is "fail if any suite has failures or errors":

```rego
package policy

import rego.v1

default allow = false

violations contains msg if {
    some suite in input.junit_results
    suite.failures > 0
    msg := sprintf("suite '%s' has %d failures", [suite.name, suite.failures])
}

violations contains msg if {
    some suite in input.junit_results
    suite.errors > 0
    msg := sprintf("suite '%s' has %d errors", [suite.name, suite.errors])
}

allow if count(violations) == 0
```

## How to use it

The pattern is identical to SARIF — see the [SARIF tutorial](/tutorials/attest_sarif) for the full walkthrough. The legacy `kosli attest junit` command (which hardcodes the "zero failures" compliance rule) continues to work; see [`kosli attest junit`](/client_reference/kosli_attest_junit) for that path.
