title: "System type: Jira"
description: The Jira system attestation type — record that a change is tracked by an existing Jira issue.
---

`system:jira` is for attesting that a code change references one or more existing Jira issues. The Kosli CLI parses your commit messages (or another source you point it at) for Jira keys, then verifies each key resolves to a real Jira issue.

## Versions

| Version | Status | Notes |
|---|---|---|
| `v1` | Current | Initial release. Default compliance is structural only — attach a rego policy to enforce thresholds like "at least one valid issue reference". |

## What the attestation accepts

A Jira attestation carries:

- **`jira_results`** *(required)* — an array of Jira issue references. Each entry includes:
  - `issue_id` — the Jira key (e.g. `PROJ-123`)
  - `issue_url` — the full URL to the issue
  - `issue_exists` — boolean; whether the CLI successfully confirmed the issue exists when the attestation was made
  - `issue_fields` *(optional)* — additional fields like summary and description, surfaced in the attestation tile
- **`evaluation_result`** *(optional)* — the JSON output of `kosli evaluate` with a rego policy. When supplied, drives compliance.
- **Policy file attachment** *(required when `evaluation_result` is supplied)* — the `.rego` file is uploaded to the audit trail.

## Compliance behaviour

Default: **structural only** — the attestation is compliant if `jira_results` is present and well-formed. Kosli does not impose an opinion on whether an empty issue list or a non-existent issue is acceptable; you do.

With a rego policy attached: the policy's `allow` flag drives compliance. A common policy is "every reference must point at a real Jira issue, and there must be at least one":

```rego
package policy

import rego.v1

default allow = false

violations contains msg if {
    count(input.jira_results) == 0
    msg := "no Jira issues were referenced"
}

violations contains msg if {
    some issue in input.jira_results
    not issue.issue_exists
    msg := sprintf("Jira issue %s does not exist", [issue.issue_id])
}

allow if count(violations) == 0
```

## How to use it

The pattern is identical to SARIF — see the [SARIF tutorial](/tutorials/attest_sarif) for the full walkthrough. The legacy `kosli attest jira` command (which hardcodes the "all issues exist" compliance rule) continues to work; see [`kosli attest jira`](/client_reference/kosli_attest_jira) for that path.
